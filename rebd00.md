# Introdução

## 1. O que o sistema faz?
O objetivo é centralizar tudo o que um cinema precisa para funcionar no dia a dia:

Gerir Filmes e Salas: Registar quais filmes estão em exibição e qual a lotação de cada sala.
Organizar Sessões: Definir horários, garantindo que não existam dois filmes ao mesmo tempo na mesma sala.
Venda de Bilhetes: Permitir que os clientes comprem bilhetes de forma rápida.

## 2. A "Liberdade de Escolha" (O grande diferencial)
Ao contrário de outros cinemas, este sistema não marca lugares.
O cliente apenas escolhe a sessão e diz quantos bilhetes quer.
Quando chegar à sala, senta-se onde quiser. Isso torna o sistema mais simples de usar e gerir.

## 3. Quem utiliza o sistema?
O sistema foi pensado para duas pessoas diferentes:

O Cliente: Entra no site, vê que filmes há, vê se ainda há lugares e compra o seu bilhete sozinho.
O Administrador: É o "chefe". Ele adiciona novos filmes, cria as sessões nas salas e controla quantos bilhetes foram vendidos para saber se o cinema está cheio.

## 4. O que o sistema NÃO faz?
Para manter o trabalho focado e simples, ficaram de fora:

Venda de pipocas ou bebidas.
Gestão de funcionários (limpeza ou atendimento).

## 5. Como é construído (A parte técnica)
Tu vais criar uma base de dados com 5 tabelas principais: Filme, Sala, Sessão, Cliente e Bilhete.

Regras: Cada tabela tem de ter pelo menos 30 exemplos de dados para testar.

Tecnologia: Vais usar Node.js para a lógica, MySQL para guardar os dados e criar um site (interface gráfica) para as pessoas interagirem.
